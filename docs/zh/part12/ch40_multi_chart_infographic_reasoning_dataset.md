# 第40章：多图表信息图推理数据工程

## 源文件

- 原始材料：`multi-chart infographic reasoning dataset.docx`

## 章节定位

本章用于把 multi-chart infographic reasoning dataset 整理为“复合信息图、多子图、多步推理”的数据工程案例。它要说明模型面对的不是单张柱状图或单个表格，而是包含多种图表、符号、图例和上下文说明的真实信息图。

本章承接第三篇的图文对、多模态理解和跨模态对齐，第七篇的证据组织与检索增强，第十二篇前两章的文档/表格结构化数据，并为第十三篇 VLM 与多模态生成数据配方提供复杂视觉推理 benchmark 案例。

## 写作目标

- 讲清数据集任务：cross-chart aggregation、multi-step calculation、visual and contextual inference。
- 讲清样本组成：354 张真实复合信息图、1,917 个多步互相关联子问题，以及多类型图表组合。
- 讲清问题链设计：从子问题到最终答案，如何体现跨图表证据聚合。
- 讲清标注和评测难点：图例解释、数值读取、单位换算、条件筛选、多步计算和错误传播。
- 讲清当前限制：原始材料说明该工作目前处在 dataset stage，尚无对应算法；正文必须如实呈现。

## 素材使用要求

- 必须保留 docx 中的数据规模、图表类型、问题类型和“尚无对应算法”的事实。
- 不要虚构模型结果、排行榜或 baseline。
- 如果补充案例，必须清楚区分“示例性重写”和“原数据集已有样本”。

## 建议结构

1. 问题场景：为什么单图表 VQA 无法覆盖真实信息图推理。
2. 数据集概览：图像规模、子问题规模、领域覆盖和图表类型。
3. 样本结构：信息图、子图、问题链、证据位置、答案和推理路径。
4. 构建流水线：图像收集、子图识别、问题链设计、答案校验、难度分层。
5. 评测协议：单步正确率、多步链路正确率、证据定位、计算一致性。
6. 失败模式：读错图例、跨图混淆、单位遗漏、早期子问题错误传播。
7. 与前后章节回链：连接 Ch08-Ch11、Ch21-Ch23、Ch38-Ch39、Ch47 和 P05。

## 篇幅要求

- 目标篇幅：12-14 页。

- 至少包含 2 张流程图、2 个表格、1 个完整问题链样例。

- 子问题链与证据聚合流程图。
- 图表类型与问题类型矩阵表。
- 多步推理错误传播分析表。

- 是否如实说明当前没有配套算法，避免夸大成果。
- 是否给出清晰的样本 schema，方便后续学生补代码或实验。
- 是否连接到 VLM、RAG 证据组织和多模态评测。
